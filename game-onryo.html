/ 01 — OVERVIEW

No body. Many bodies.

You start as a pure spirit — no body, no weapons, no way to fight directly. To survive in the arena, you must possess enemies and use their bodies against each other. Every enemy type plays differently: their weapons, stats, and behavior are their own.

Built over 2-3 months by a team of 6 in Unreal Engine 5 with C++. I touched almost every part of the codebase — possession system, FPS mechanics, character controllers, UI/HUD, and enemy AI integration.

/ 04 — ARCHITECTURE

How it's built.

Possession — shared logic, split responsibilities

The core design challenge was: how do you let the player inhabit any enemy without duplicating or branching logic for every character type?

The answer was a clean split of responsibilities. Everything common to both the player and any possessed enemy — movement, jump, look, possess, unpossess — lives in the PlayerController. It's always available the moment possession happens, regardless of which body is being controlled.

Everything specific to an enemy type — their particular attack, ability, or behavior — is implemented directly in the enemy's Pawn. This way, the same Pawn works under two control sources without branching: the PlayerController drives it when possessed, the AIController drives it when not. Shared logic stays in one place, specific logic stays where it belongs.

Enemy behavior — shared structure, different logic

Each enemy type has its own state machine, which runs whether the enemy is AI-controlled or player-possessed. This keeps behavior rules consistent and avoids duplicating code across control sources.

The two enemy types — Base and Sentinel — share the same exit conditions (dead or player dead) and stun state, but differ in their core loop. The Base actively pursues the player: it focuses, moves toward them, and shoots when in range. The Sentinel first moves to a fixed patrol spot before locking onto and engaging the player from position. Both can be interrupted by a stun at any point in their loop.

/ 05 — TECH HIGHLIGHTS

Under the hood.

  • 01UE5 + C++ — full gameplay architecture, 95.7% C++.
  • 02Possession via PlayerController transfer — no logic duplication across character types.
  • 03Dual control source — same character class works under AIController or PlayerController.
  • 04Per-enemy state machines — Base (pursue & shoot) and Sentinel (patrol then engage).
  • 05Shared stun state — interruptible at any point in the enemy behavior loop.
  • 06FPS mechanics — responsive movement, shooting, melee, per-class tuning in C++.
  • 07UI / HUD — in-game interface reflecting current possessed body and state.
