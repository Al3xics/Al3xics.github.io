/ 01 — OVERVIEW

Revive a dying world.

The world is broken. You play as an avatar of the world itself — born from what remains. Your tools: mushrooms that spread life across a board of tiles, puzzles that unlock the land, and a narrative that unfolds level by level.

Built in Unreal Engine 5 with C++ by a team of 9 people. The project started 3-4 months ago and will run for roughly a year total — we're currently at the halfway point, in the polishing phase with active bug fixing on the programming side.

I worked on the gameplay mechanics, UI/HUD, the puzzle system, the wave propagation system, board generation and tile editing tools for designers, and the player's movement in and out of the board.

/ 05 — ARCHITECTURE

How it's built.

Board generation & designer tools

The board is generated programmatically and exposed to designers through custom in-editor tools. Designers can create and configure the board, then modify individual tiles directly in the Unreal editor to shape each level — without touching code. I built both the generation system and the tile editing tools that make this workflow possible.

// Board generation — grid creation via custom editor tools

Wave propagation system

The core mechanic I'm most proud of is the wave propagation system. When a tile receives an input — from a player action for instance — it evaluates whether its propagation rule is triggered. If so, it sends the new tile type to each of its neighbors. Each neighbor then runs the same check and forwards the wave further. This continues recursively until no more neighbors satisfy the rule.

The same mechanism runs at level completion: when the player wins, the propagation spreads across all untransformed tiles on the board, creating a visual wave of revival that fills the entire level.

The system is designed to be scalable — adding a new tile type or propagation rule doesn't require touching existing logic, just extending it.

// Left: player turn & tile placement loop — Center: wave propagation per neighbor — Right: tile type change trigger

Game loop — turn order & win/lose conditions

When the player clicks a tile, the propagation loop begins. Each wave type is processed in a fixed order: Grass → Enemy → Collectible → Water. For each wave, all changes are first computed from the origin tile outward — nothing is applied yet. Once the full wave is calculated, the changes are applied. Then the next wave type runs using the same compute-then-apply pattern. This loop repeats continuously — after Water, it cycles back to Grass — until a full wave produces zero changes. For example: Grass propagates and modifies tiles, changes are applied, Enemy wave runs but nothing triggers, the loop stops there. No wasted computation, and the system is naturally self-terminating. Player movement in and out of the board is handled separately, allowing free navigation between the overworld and the puzzle board.
